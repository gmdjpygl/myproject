多窗口播放视频 NPPILY.PlayVideo

当页面加载完成，获取资源之后，就可以播放视频了。播放视频之前首先激活播放窗口容器，然后创建窗口插件。需要在调用播放视频接口之前，先 调用NPPILY.CreateWindow接口。该接口会在选择的窗口容器中创建窗口插件，返回窗口信息，并记录在NPPILY.Struct.WindowStruct，将每次记录的 NPPILY.Struct.WindowStruct保存在NPPILY.Struct.WindowContainerStruct，以便在不同的视频窗口容器找到对应的窗口信息。播放视频前先遍历窗口 容器信息，匹配对应播放窗口。接着调用播放视频接口NPPILY.PlayVideo,就可以播放视频了。具体请看下面的接口说明。

窗口容器信息结构 NPPILY.Struct.WindowContainerStruct

变量 类型 描述
container Object NOT NULL 窗口插件的Dom容器对象
type NPPILY.Enum.WindowTypeNOT NULL 窗口类型
active Bool NULL 窗口是否被激活
window NPPILY.Struct.WindowStruct NULL 窗口对象，保存NPPILY.CreateWindow创建的窗口对象
description Object 正在播放的视频资源描述

窗口事件信息结构 NPPILY.Struct.WindowEventStruct

变量 类型 描述
lbtn_click Object({status: true_or_false, callback: function () {...}?}) NULL 鼠标左键点击
select_rect Object({status: true_or_false, callback: function () {...}?}) NULL 矩形框选择
ptz_control Object({status: true_or_false, callback: function () {...}?}) NULL 云台控制
fsw_show Object({status: true_or_false, callback: function () {...}?}) NULL 显示全屏（建议status属性值不用设置，因为无论如何双击视频窗口，均会全屏的）
fsw_hide Object({status: true_or_false, callback: function () {...}?}) NULL 关闭全屏
menu_command Object({status: true_or_false, menu: [{key: ?, text: ?}, ...], callback: function () {...}?}) NULL 自定义右键菜单项

示例：


JavaScript源码： 

			
 一、视频播放
 	
	// - 添加窗口事件
	var windowAttachEvent = new NPPILY.Struct.WindowEventStruct();
	windowAttachEvent.lbtn_click.status = true;
	windowAttachEvent.lbtn_click.callback = function() {
		// 鼠标左键单击响应回调函数
	}
	windowAttachEvent.select_rect.status = true;
	windowAttachEvent.select_rect.callback = function() {
		/*鼠标拉选响应回调函数,注意在窗口模式为“框选”时有效，
		  回调时callback(x1, y1, x2,y2)，其中(x1,y1)、(x2,y2)
		  分别代表窗口中框选的左上与右下坐标*/
	}
	windowAttachEvent.ptz_control.status = true;
	windowAttachEvent.ptz_control.callback = function() {
		// 窗口云台控制响应回调函数，内部会自行处理云台控制，上层不需要发控制命令
	}
    // fsw_show.status | fsw_hide.status置为false没有意义，造成全屏黑屏
	windowAttachEvent.fsw_show.status = true;
	windowAttachEvent.fsw_show.callback = function() {
		// 窗口全屏后的回调函数
	}
	windowAttachEvent.fsw_hide.status = true;
	windowAttachEvent.fsw_hide.callback = function() {
		// 窗口全屏恢复后的回调函数
	}
    // 自定义视频窗口右键菜单项
    windowAttachEvent.menu_command.status = true;
    windowAttachEvent.menu_command.menu = [{
        key: "stopvideo", text: "停止视频"							
    }, {
        key: "playaudio", text: "播放音频"
    }, {
        key: "localsnapshot", text: "本地抓拍"
    }, {
        key: "localrecord", text: "开启本地录像"
    }, {
        key: "separator", text: "first"							
    }, {
        key: "fullscreen", text: "全屏"		
    }];
    windowAttachEvent.menu_command.callback = function (menukey) {
        // 选择右键菜单项回调，menukey对应上面的key，如menukey == "stopvideo"
    };
	
	// 在播放视频之前，应先调用创建窗口
	var operator = NPPILY.CreateWindow(connectId, windowkey, NPPILY.Enum.WindowType.VIDEO, windowAttachEvent);
	if(operator.rv == NrcapError.NRCAP_SUCCESS) {
    	// wnd为窗口信息，应该记录到NPPILY.WindowContianers中
		var wnd = operator.response;
		// 播放视频
		var play_operator = NPPILY.PlayVideo(connectId, wnd, puid, idx, NPPILY.Enum.NrcapStreamType.REALTIME);
		if(play_operator.rv == NrcapError.NRCAP_SUCCESS) {
			// 播放成功
		}
	}

二、喊话对讲
	
    1）获取资源，在线的设备，判断设备子资源中是否含有音频输出资源（资源类型是NPPILY.Enum.PuResourceType.AudioOut）
    2）喊话对讲针对的是设备，与视频资源没有关系，不应该搞对应操作，音频输出资源模块数一个设备一般为一个，也可能没有（此时肯定不支持喊话对讲）
    3）开启喊话为NPPILY.CallTalkControl.StartCall(connectId, puid, oa_res_idx);puid为设备的PUID，oa_res_idx为音频输出资源索引号
    4）开启对讲为NPPILY.CallTalkControl.StartTalk(connectId, puid, oa_res_idx);注意同个设备喊话对讲操作是互斥的
    5）停止喊话对讲函数是一样的NPPILY.CallTalkControl.Stop(connectId, puid, oa_res_idx);
    6）获取喊话对讲状态函数var oaStatus_operator = NPPILY.CallTalkControl.GetStatus(connectId, puid, oa_res_idx);此接口很有用，在开启喊话对讲前后调用即可知晓状态，便于上层代码处理；oaStatus_operator.response结构为 
        {
        	puid: puid, 
            oaIndex: oa_res_idx, 
            call: true_or_false, // 是否在喊话
			talk: true_or_false, // 是否在对讲
            oaStreamHandle: ? // 喊话对讲句柄
        }
        其中oaStreamHandle为空，表示没有喊话对讲，不为空说明有喊话对讲
    7) 获取所有正在喊话和对讲状态的信息NPPILY.CallTalkControl.GetAllRunningStatus(connectId);返回值response节点是数组结构如下：
    	[{puid: ?, oaIndex: ?, call: ?, talk: ?, oaStreamHandle: ?}, ...]
